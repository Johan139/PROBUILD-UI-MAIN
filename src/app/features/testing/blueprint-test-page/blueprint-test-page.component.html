Apollo Direct API Testing (No Backend Persistence)

This test tab calls Apollo directly from the browser. Use for exploratory testing only.

How this test works (important)
  • Requests are sent directly from your browser to Apollo endpoints.
  • No data from this tab is persisted to ProBuild backend storage.
  • Use this for exploratory validation and payload inspection only.
  • Successful responses are shown as formatted JSON in the two response panels below.
  • Errors (401/403/422/429/etc.) are rendered in the error block above each response panel.
Field-by-field input guide
  • Apollo API Key: Required for both actions. Paste your valid Apollo secret key. This is applied as X-Api-Key and Authorization: Bearer ....
  • Company Name: Company/org name hint used in keyword search and enrich fallback. Example: ACME Construction Ltd.
  • Domain (optional): Preferred for enrich calls when available. Example: acme.com (no protocol needed).
  • Trade: Trade keyword to guide subcontractor discovery. Example values: Electrical, Plumbing, Drywall, Roofing.
  • City: Geographic search hint. Example: London.
  • State / Region: Region hint to narrow discovery. Example: Greater London, Texas.
  • Search limit: Max records for mixed_companies/search. Recommended range: 5-25 for quick tests.
Which button to use and when
  • Run mixed_companies/search: broad discovery flow. Builds a keyword query from company/trade/city/state + subcontractor.
  • This tester also applies Apollo query params from docs for discovery quality: organization_locations[] and q_organization_keyword_tags[].
  • Run organizations/enrich: organization enrichment flow. Uses Domain when provided, otherwise uses Company Name.
  • Clear Results: clears both response and error boxes so you can run clean tests.
Recommended test workflow
  1. Paste Apollo API key.
  2. Set trade + location hints first (trade/city/state).
  3. Run mixed_companies/search and inspect returned organizations.
  4. Take one returned domain or organization name and place it in Domain/Company Name.
  5. Run organizations/enrich and compare detail quality.
  6. Use Clear Results between test cases for cleaner diffing/screenshots.
